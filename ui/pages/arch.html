系统架构图

这里会沉淀多张架构图。当前先重画“智能问数”主链路，展示从自然语言输入到 SQL 生成、执行、反馈学习的完整判断逻辑。

智能问数 Query-to-SQL 调用链路

默认入口为 Cube 语义层。LangChain / Vanna 保留为手动切换或历史兼容路径。

主链路 缓存/召回 判断逻辑 安全拦截
00

用户入口层

前端问数工作台通过 SSE 调用当前选择的引擎，默认 `Cube`。

GET /ask/cube/stream POST /ask/cube EventSource Trace ID
页面先预渲染 7 个调用步骤卡片，后端逐步推送 `start / step_start / step_done / final / error`。
01

自然语言安全守卫

先判断用户问题是否包含破坏性意图，命中后立即拒绝，不进入模型、RAG 或数据库。

删除/清空/修改 DROP / UPDATE / INSERT 只读问数
示例：`删除销售表` 会返回违规提示；不会再被误解析成 `SELECT AVG(...)`。
02A

Cube 模型检查

加载 Doris 中维护的 Cube 模型，并渲染为内存 Bundle。

cube_models cube_measures cube_dimensions cube_joins cube_dimension_values
配置开关控制加载模式：默认首次加载到内存；开启后每次请求都从 Doris 重读。
02B

内存缓存同步

管理页保存 Cube 实体后，可点击“同步缓存”将最新配置加载到运行时。

/cube/reload-model Metric Cube 管理页 枚举采集后自动刷新
如果 `cube_dimension_values` 不存在，系统会降级使用 `cube_dimensions.enum_mapping_json`，不阻断问数。
03A

意图理解 Agent

Qwen 解析用户问题，输出标准化问题、业务域、复杂度、时间提示。

intent_type business_domain complexity time_hint
与 SQL RAG 并行执行，减少总耗时。时间表达会交给 TimeScopeCompiler 做确定性补强。
03B

SQL RAG 召回

对参数占位后的 canonical question 做向量检索，召回已验证正确 SQL。

semantic_store.vanna_semantic_sql_rag score = 1 - distance quality_score
每条召回样本在调用链路里展示 `score / dist / quality`，用于排查召回质量。
04

Prompt 构建与 Cube 语义解析

把 Cube measures、dimensions、segments、RAG 示例、时间提示注入 Prompt，要求 LLM 只返回结构化 JSON，不返回 SQL。

measures[] dimensions[] filters[] segments[] order limit
如果命中规则模板，例如“4月份各城市销售额”，可跳过 LLM 解析，直接进入规则计划。
05

规则修正与枚举识别

对 LLM 结果做确定性修正，补时间、补常见指标、识别城市/会员/类目等枚举筛选。

TimeScopeCompiler enum_mapping dimension_values category_1_id member_type
例：`北京地区 PLUS会员销售额` 会识别为 `city_code=北京`、`member_type=PLUS会员`，再映射成真实过滤值。
06

Cube SQL 编译器

根据 Cube 模型生成 Doris 可执行 SQL：自动解析主 Cube、关联 Cube、JOIN、WHERE、GROUP BY、ORDER BY。

_compile_sql _resolve_joins _render_filter _map_enum_value
判断逻辑：以第一个 measure 的 cube 为主表；维度/过滤/segment 引用其他 cube 时必须存在 join 配置，否则直接报错。
07

SQL 只读安全校验

最终 SQL 必须是单条 `SELECT` 或 `WITH ... SELECT`，禁止 DDL/DML/权限/导入导出等操作。

assert_readonly_sql SELECT / WITH DROP/DELETE 拦截 单语句限制
这是最终防线；即使 LLM 或模板异常生成危险 SQL，也不会进入执行层。
08A

Doris 执行层

只读 SQL 通过 PyMySQL 连接 Doris，结果转成表格 JSON 返回前端。

retail_dw query_df execute
前端“执行 SQL”也复用同一套只读校验，支持 CTE。
08B

Trace 与审计日志

每个步骤记录输入、输出、耗时、错误和最终 SQL，持久化到 `vanna_store.vanna_trace_log`。

trace_id step_done total_ms 调用日志近 30 条
Cube trace 连接采用懒加载，只在写日志时连接 `vanna_store`。
09

反馈学习闭环

用户点击“正确，加入知识库”后，Cube/Semantic 写入 `vanna_semantic_sql_rag`，同时保留 Vanna 训练样本。

feedback canonicalize_question 去重 stable_rag_id Embedding 入库
下次类似问题只要参数变化，例如北京换上海，会因 canonical question 命中同一类模板而提升召回。
