① Intent Understanding Agent
调用 Qwen LLM 解析意图，输出结构化 IntentPlan
intent_type business_domain time_hint entity_hints compare_type
输出字段
· intent_type：metric_query / metric_compare / trend / ranking / detail
· business_domain：sales / refund / user / product 等
· time_hint：解析出的时间范围 / 粒度
· entity_hints：提到的实体（大区、渠道、品类）
· compare_type：mom / yoy / custom

实现：vanna_skill/semantic/agents/intent_agent.py

Coverage Score 计算
score = (指标命中数 + 维度命中数) / 请求总数

· ≥ 0.8：全部命中，走确定性模板 SQL，无需 LLM
· 0.5-0.8：部分命中，语义上下文注入 LLM 辅助生成
· < 0.5：未能识别，降级到 LangChain 向量检索路径

注意：当前评分仅检查 catalog 存在性，尚未校验 metric-dimension 兼容矩阵。

SQL Synthesizer
无 LLM 确定性模板合成
{alias} 占位符替换
SELECT FROM + JOIN WHERE / GROUP BY
合成步骤
1. 从第一个 MetricDef 取主表 + 别名
2. 收集 extra_joins + dimension.join，去重
3. 替换 {alias} / {fact_alias} / {time_col} 占位符
4. 构建 SELECT（维度 + 指标聚合表达式）
5. 构建 WHERE（时间范围优先 + filters）
6. 构建 GROUP BY / ORDER BY / LIMIT

实现：vanna_skill/semantic/sql_synthesizer.py
方言：Apache Doris（预留 MySQL 扩展点）

Few-shot SQL 生成
向量检索 SQL 示例 文档召回
降级流程
1. 对用户问题做 Embedding（Qwen text-embedding-v3）
2. 在 vanna_store.vanna_collection 中做向量相似度召回
3. 取 Top-N SQL 示例 + Schema + Docs 组装 Few-shot Prompt
4. Qwen LLM 生成最终 SQL

注意：此路径完全绕过 semantic_store，依赖 vanna_store 中的训练数据质量。

实现：vanna_skill/pipelines/ask_lc_pipeline.py
