## 一、系统定位与目标

| 目标 | 说明 |
|---|---|
| 自然语言查询 | 业务人员用中文提问，系统自动生成 SQL 并返回结果 |
| 零运维知识库 | Doris 自身的 information_schema / audit_log 作为训练来源，无需额外维护 |
| 全链路闭环 | 用户反馈→向量库自动更新→下次命中率提升 |
| 无额外中间件 | Doris 同时承担业务库、向量库、元数据库三种角色，无需 Milvus |

## 三、Doris 向量知识库设计（替代 Milvus）

Doris 2.1+ 原生支持 `ARRAY<FLOAT>` 向量列和 `cosine_distance()` 函数，完全可替代 Milvus。

### 3.1 向量存储主表

```sql
CREATE DATABASE IF NOT EXISTS vanna_store;
USE vanna_store;

CREATE TABLE IF NOT EXISTS vanna_embeddings (
    id              BIGINT          NOT NULL AUTO_INCREMENT  COMMENT '主键',
    content_type    VARCHAR(20)     NOT NULL                 COMMENT 'sql / ddl / doc',
    question        TEXT                                     COMMENT '自然语言问题（sql类型必填）',
    content         TEXT            NOT NULL                 COMMENT 'SQL语句 / DDL定义 / 文档片段',
    embedding       ARRAY<FLOAT>    NOT NULL                 COMMENT '768维向量（Gemini text-embedding-004）',
    source          VARCHAR(50)                              COMMENT 'audit_log / manual / schema / feedback',
    db_name         VARCHAR(100)                             COMMENT '所属数据库',
    table_names     TEXT                                     COMMENT '涉及的表名（逗号分隔）',
    use_count       INT             DEFAULT 0                COMMENT '历史命中次数',
    quality_score   FLOAT           DEFAULT 1.0              COMMENT '质量评分（0-1）',
    created_at      DATETIME        DEFAULT NOW()            COMMENT '入库时间',
    updated_at      DATETIME        DEFAULT NOW()            COMMENT '更新时间'
) ENGINE=OLAP
UNIQUE KEY(id)
COMMENT 'Vanna 向量知识库'
DISTRIBUTED BY HASH(id) BUCKETS 4
PROPERTIES ("replication_num" = "1");
```

### 3.2 用户反馈表（支持持续学习）

```sql
CREATE TABLE IF NOT EXISTS vanna_feedback (
    id              BIGINT          NOT NULL AUTO_INCREMENT,
    question        TEXT            NOT NULL  COMMENT '用户原始问题',
    generated_sql   TEXT            NOT NULL  COMMENT '系统生成的 SQL',
    corrected_sql   TEXT                      COMMENT '用户修正后的 SQL（如有）',
    is_correct      BOOLEAN         NOT NULL  COMMENT '结果是否正确',
    embedding_id    BIGINT                    COMMENT '对应向量库记录',
    session_id      VARCHAR(64)               COMMENT '会话ID',
    created_at      DATETIME        DEFAULT NOW()
) ENGINE=OLAP
UNIQUE KEY(id)
DISTRIBUTED BY HASH(id) BUCKETS 2
PROPERTIES ("replication_num" = "1");
```

### 3.3 向量检索 SQL（核心查询）

```sql
-- 按余弦相似度检索最近似的 question-sql 对
SELECT
    id, question, content AS sql_text,
    cosine_distance(embedding, CAST(? AS ARRAY<FLOAT>)) AS distance
FROM vanna_store.vanna_embeddings
WHERE content_type = 'sql'
  AND quality_score >= 0.5
ORDER BY distance ASC
LIMIT 5;
```

### 4.3 audit_log SQL 挖掘（质量过滤规则）

```python
AUDIT_SQL_QUERY = """
SELECT stmt, query_time, scan_bytes, return_rows
FROM __internal_schema.audit_log
WHERE 1=1
  AND stmt       LIKE 'SELECT%'          -- 只取查询语句
  AND state      = 'EOF'                  -- 执行成功
  AND query_time < 30000                  -- 30秒内完成
  AND scan_bytes < 10737418240            -- 扫描量 < 10GB
  AND return_rows BETWEEN 1 AND 100000    -- 有结果但不过大
  AND stmt NOT LIKE '%vanna_store%'       -- 排除系统查询
ORDER BY query_time ASC
LIMIT 2000
"""

QUALITY_FILTERS = [
    lambda sql: len(sql) > 30,                    # 过滤过短SQL
    lambda sql: 'information_schema' not in sql,  # 过滤系统查询
    lambda sql: sql.count('JOIN') <= 5,           # 过滤过于复杂SQL
]
```

## 五、Vanna 自定义实现（DorisVanna）

```python
# vanna_doris/doris_vanna.py

from vanna.base import VannaBase
import pymysql
import google.generativeai as genai
from typing import List, Tuple, Union
import json

class DorisVanna(VannaBase):
    """
    Vanna 的 Doris 后端实现：
    - Vector Store: Doris vanna_store 库
    - LLM + Embedding: Gemini (text-embedding-004 + gemini-2.0-flash)
    - Data Source: retail_dw (或任意 Doris 业务库)
    """

    def __init__(self, config: dict):
        super().__init__(config=config)
        # Gemini 初始化
        genai.configure(api_key=config['gemini_api_key'])
        self.llm = genai.GenerativeModel(
            config.get('model', 'gemini-2.0-flash')
        )
        # Doris 连接（向量库）
        self._vec_conn = pymysql.connect(
            host=config['host'], port=config['port'],
            user=config['user'], password=config.get('password', ''),
            database='vanna_store', charset='utf8mb4'
        )
        # Doris 连接（业务库）
        self._biz_conn = pymysql.connect(
            host=config['host'], port=config['port'],
            user=config['user'], password=config.get('password', ''),
            database=config.get('database', 'retail_dw'), charset='utf8mb4'
        )
        self.n_results = config.get('n_results', 5)

    # ──────────────────────────────────────────────
    # Embedding
    # ──────────────────────────────────────────────
    def generate_embedding(self, data: str, **kwargs) -> List[float]:
        result = genai.embed_content(
            model="models/text-embedding-004",
            content=data,
            task_type="retrieval_document"
        )
        return result['embedding']  # 768维 float list

    # ──────────────────────────────────────────────
    # SQL 生成（LLM核心）
    # ──────────────────────────────────────────────
    def submit_prompt(self, prompt, **kwargs) -> str:
        response = self.llm.generate_content(prompt)
        return response.text

    # ──────────────────────────────────────────────
    # 写入：question-sql 对
    # ──────────────────────────────────────────────
    def add_question_sql(self, question: str, sql: str, **kwargs) -> str:
        embedding = self.generate_embedding(question)
        source = kwargs.get('source', 'manual')
        with self._vec_conn.cursor() as cur:
            cur.execute("""
                INSERT INTO vanna_embeddings
                    (content_type, question, content, embedding, source, db_name)
                VALUES ('sql', %s, %s, %s, %s, %s)
            """, (question, sql, json.dumps(embedding), source,
                  self._biz_conn.db.decode()))
        self._vec_conn.commit()
        return "ok"

    # ──────────────────────────────────────────────
    # 写入：DDL
    # ──────────────────────────────────────────────
    def add_ddl(self, ddl: str, **kwargs) -> str:
        embedding = self.generate_embedding(ddl)
        with self._vec_conn.cursor() as cur:
            cur.execute("""
                INSERT INTO vanna_embeddings
                    (content_type, content, embedding, source, db_name)
                VALUES ('ddl', %s, %s, 'schema', %s)
            """, (ddl, json.dumps(embedding), self._biz_conn.db.decode()))
        self._vec_conn.commit()
        return "ok"

    # ──────────────────────────────────────────────
    # 写入：文档
    # ──────────────────────────────────────────────
    def add_documentation(self, documentation: str, **kwargs) -> str:
        embedding = self.generate_embedding(documentation)
        with self._vec_conn.cursor() as cur:
            cur.execute("""
                INSERT INTO vanna_embeddings
                    (content_type, content, embedding, source)
                VALUES ('doc', %s, %s, 'doc')
            """, (documentation, json.dumps(embedding)))
        self._vec_conn.commit()
        return "ok"

    # ──────────────────────────────────────────────
    # 检索：相似 question-sql 对
    # ──────────────────────────────────────────────
    def get_similar_question_sql(self, question: str, **kwargs) -> List[Tuple]:
        embedding = self.generate_embedding(question)
        with self._vec_conn.cursor() as cur:
            cur.execute("""
                SELECT question, content,
                       cosine_distance(embedding, CAST(%s AS ARRAY<FLOAT>)) AS dist
                FROM vanna_embeddings
                WHERE content_type = 'sql'
                  AND quality_score >= 0.5
                ORDER BY dist ASC
                LIMIT %s
            """, (json.dumps(embedding), self.n_results))
            rows = cur.fetchall()
            # 更新命中计数
            for row in rows:
                cur.execute("""
                    UPDATE vanna_embeddings
                    SET use_count = use_count + 1
                    WHERE question = %s AND content_type = 'sql'
                """, (row[0],))
        self._vec_conn.commit()
        return [(r[0], r[1]) for r in rows]  # (question, sql)

    # ──────────────────────────────────────────────
    # 检索：相关 DDL
    # ──────────────────────────────────────────────
    def get_related_ddl(self, question: str, **kwargs) -> List[str]:
        embedding = self.generate_embedding(question)
        with self._vec_conn.cursor() as cur:
            cur.execute("""
                SELECT content,
                       cosine_distance(embedding, CAST(%s AS ARRAY<FLOAT>)) AS dist
                FROM vanna_embeddings
                WHERE content_type = 'ddl'
                ORDER BY dist ASC
                LIMIT %s
            """, (json.dumps(embedding), self.n_results))
            return [r[0] for r in cur.fetchall()]

    # ──────────────────────────────────────────────
    # 检索：相关文档
    # ──────────────────────────────────────────────
    def get_related_documentation(self, question: str, **kwargs) -> List[str]:
        embedding = self.generate_embedding(question)
        with self._vec_conn.cursor() as cur:
            cur.execute("""
                SELECT content,
                       cosine_distance(embedding, CAST(%s AS ARRAY<FLOAT>)) AS dist
                FROM vanna_embeddings
                WHERE content_type = 'doc'
                ORDER BY dist ASC
                LIMIT 3
            """, (json.dumps(embedding),))
            return [r[0] for r in cur.fetchall()]

    # ──────────────────────────────────────────────
    # 执行 SQL
    # ──────────────────────────────────────────────
    def run_sql(self, sql: str) -> any:
        import pandas as pd
        return pd.read_sql(sql, self._biz_conn)

    # ──────────────────────────────────────────────
    # 获取所有训练数据（供调试）
    # ──────────────────────────────────────────────
    def get_training_data(self, **kwargs):
        import pandas as pd
        return pd.read_sql(
            "SELECT id, content_type, question, content, source, use_count "
            "FROM vanna_embeddings ORDER BY created_at DESC LIMIT 200",
            self._vec_conn
        )

    def remove_training_data(self, id: str, **kwargs) -> bool:
        with self._vec_conn.cursor() as cur:
            cur.execute("DELETE FROM vanna_embeddings WHERE id = %s", (id,))
        self._vec_conn.commit()
        return True
```

```python
# agent/qa_agent.py
from langchain_google_genai import ChatGoogleGenerativeAI
from langchain.agents import AgentExecutor, create_react_agent
from langchain.memory import ConversationBufferWindowMemory
from langchain import hub

SYSTEM_PROMPT = """你是一个专业的数据分析助手，能将业务问题转化为 Doris SQL 查询并解读结果。

工作流程：
1. 理解用户问题的业务含义
2. 如需要，调用 schema_query 了解表结构
3. 调用 text_to_sql 生成 SQL
4. 调用 execute_sql 执行并获取数据
5. 用自然语言解读查询结果，给出业务洞察

注意：
- 生成的 SQL 只能是 SELECT 语句，不得修改数据
- 如果问题不清晰，主动追问澄清
- 对敏感数据（用户手机号等）提醒注意脱敏
"""

def build_agent(config: dict):
    llm = ChatGoogleGenerativeAI(
        model="gemini-2.0-flash",
        google_api_key=config['gemini_api_key'],
        temperature=0
    )
    memory = ConversationBufferWindowMemory(
        memory_key="chat_history",
        k=10,                    # 保留最近10轮对话
        return_messages=True
    )
    vanna = DorisVanna(config)
    tools = make_tools(vanna, ...)
    
    agent = create_react_agent(llm, tools, prompt=SYSTEM_PROMPT)
    return AgentExecutor(
        agent=agent,
        tools=tools,
        memory=memory,
        verbose=True,
        max_iterations=6,
        handle_parsing_errors=True
    )
```

## 七、训练初始化脚本

```python
# scripts/train_init.py
"""
初始化 Vanna 知识库：
1. 从 information_schema 加载所有表的 DDL
2. 从 audit_log 挖掘高质量历史 SQL
运行一次即可，后续增量更新
"""

def init_training(vanna: DorisVanna, db_name: str = 'retail_dw'):
    # Step 1: 加载 DDL
    print("Step 1/2: 加载表结构...")
    rows = vanna.run_sql(SCHEMA_QUERY.format(db_name=db_name))
    ddl_map = {}
    for _, row in rows.iterrows():
        tname = row['TABLE_NAME']
        if tname not in ddl_map:
            ddl_map[tname] = f"-- {row['TABLE_COMMENT']}\nCREATE TABLE {tname} (\n"
        comment = f" COMMENT '{row['COLUMN_COMMENT']}'" if row['COLUMN_COMMENT'] else ''
        ddl_map[tname] += f"  {row['COLUMN_NAME']} {row['DATA_TYPE']}{comment},\n"
    for tname, ddl in ddl_map.items():
        vanna.add_ddl(ddl.rstrip(',\n') + "\n);")
        print(f"  ✓ {tname}")

    # Step 2: 挖掘 audit_log
    print("\nStep 2/2: 挖掘历史 SQL...")
    audit_rows = vanna.run_sql(AUDIT_SQL_QUERY)
    added = 0
    for _, row in audit_rows.iterrows():
        sql = row['stmt'].strip()
        if all(f(sql) for f in QUALITY_FILTERS):
            # 用 Gemini 自动生成对应的「问题」描述
            question = vanna.generate_question(sql)
            if question:
                vanna.add_question_sql(question, sql, source='audit_log')
                added += 1
    print(f"  ✓ 从 audit_log 入库 {added} 条 SQL")
    print("\n初始化完成！")
```

## 九、完整问答流程图

```
用户提问: "上个月华东区哪些商品销售额超过10万？"
         │
         ▼
[LangChain Agent - Gemini]
         │
         ├─► schema_query("dws_sku_trade_daily")
         │       └── 返回字段: dt/sku_id/store_id/sale_qty/sale_amt
         │
         ├─► text_to_sql("上个月华东区哪些商品销售额超过10万？")
         │       │
         │       ├── Doris 向量检索: 相似问题 × 5 条
         │       ├── Doris 向量检索: 相关 DDL × 3 张
         │       ├── 组装 Prompt → Gemini 生成 SQL
         │       │
         │       └── 返回 SQL:
         │             SELECT s.sku_name, SUM(d.sale_amt) AS total_amt
         │             FROM dws_sku_trade_daily d
         │             JOIN dim_sku_info s ON d.sku_id = s.sku_id
         │             JOIN dim_store_info st ON d.store_id = st.store_id
         │             WHERE d.dt BETWEEN '2026-03-01' AND '2026-03-31'
         │               AND st.region_id = (SELECT region_id FROM ...)
         │             GROUP BY s.sku_name
         │             HAVING total_amt > 100000
         │             ORDER BY total_amt DESC
         │
         ├─► execute_sql(上述SQL)
         │       └── 返回: [{sku_name: "iPhone 15", total_amt: 320000}, ...]
         │
         ▼
[Gemini 解读结果]
"华东区上月共有 12 款商品销售额超过 10 万元，其中 iPhone 15 以 32 万元排名第一..."
         │
         ▼
用户确认结果正确
         │
         ▼
[save_feedback] → 写入 vanna_embeddings（下次相似问题更快命中）
```

## 十二、关键技术决策说明

| 决策点 | 选择 | 理由 |
|---|---|---|
| 向量库 | Doris 原生 | 无需维护 Milvus，减少运维成本，数据不出 Doris |
| Embedding 模型 | Gemini text-embedding-004 | 768维，中文支持好，与 LLM 同源 |
| LLM | Gemini 2.0 Flash | 速度快，中文理解强，SQL生成质量高 |
| 向量检索 | cosine_distance() | Doris 内置函数，简单可靠 |
| 训练来源 | 四路自动/半自动 | 冷启动靠 schema，热启动靠 audit_log，精调靠人工+反馈 |
| 会话记忆 | LangChain WindowMemory | 保留最近10轮，避免 context 过长 |
| Agent 类型 | ReAct | 推理透明，适合 SQL 生成+验证场景 |
